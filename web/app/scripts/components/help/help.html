Help

For more information on the EODIP collaboration node, please see the 'About' section of this site. If you think you can contribute ideas to the development of the node, then please click on the 'feedback' or 'contact' links on the Homepage.

Users should be aware that this is not yet a production system and consequently, no end user support is available. This node currently serves solely to support a number of EODIP projects through data provision and access. However the scope of the node’s service is likely to expand over the year.

Cookies

Cookies are used on this website.

When we provide services, we want to make them easy, useful and reliable. Where services are delivered on the internet, this sometimes involves placing small amounts of information on your device, for example, computer or mobile phone. These include small files known as cookies. They cannot be used to identify you personally.

These pieces of information are used to improve services for you through:

  • Enabling a service to recognise your device so you don't have to give the same information several times during one task
  • Measuring how many people are using services, so they can be made easier to use and there's enough capacity to ensure they are fast

You can manage these small files yourself and learn more about them through Internet browser cookies - what they are and how to manage them

Using the node

At present, the data contained within this node may only be viewed through this site, using the mapper facilities. It will very soon be able to be downloaded from this site; the download function is scheduled for imminent release.

The data contained within this site can also be used within GI software, through web feature services (WFS) or web mapping services (WMS).

QGIS

  1. Select one of the following:
    1. Add WFS Layer from a Server
    2. Add WCS Layer from a Server
    3. Add WMS Layer from a Server
  3. Give the connection a title e.g. 'Pilot collaboration node'
  4. Enter the URL:
    1. For WFS - {{wfs_capabilities_url}}
    2. For WCS - {{wcs_capabilities_url}}
    3. For WMS - {{wms_capabilities_url}}
  6. Click 'Connect' (you don't need username or password)
  7. Select the dataset of interest, then click 'Add'

ArcMap

  1. Select: 'File' – 'Add Data' – 'Add Data', then one of the following:
    1. Add WCS Layer from a Server
    2. Add WMS Layer from a Server
    3. Add WMTS Layer from a Server
  3. Enter the following URL:
    1. For WCS - {{wcs_capabilities_url}}
    2. For WMS - {{wms_capabilities_url}}
    3. For WMTS - {{wmts_capabilities_url}}
  5. Click 'Connect' (you don't need username or password)
  6. Drill down through the new connection link using 'Add'

It is intended that users will soon be able to set-up processing within the node, rather than downloading the data for local processing. There are also plans for large scale data processing through node to node transfer, for example for processing all of the available satellite data on a regular schedule.
